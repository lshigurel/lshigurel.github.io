Windows日志分析

常见事件ID

  • 安全事件-成功登录账户（4624）

  • 安全事件-账户登录失败（4625）
  • 安全事件-已注销账户（4634）、计算机试图验证账户的凭据（4776）
暴力破解事件
  • 短时间内产生大量4625日志记录
  • 存在4624->爆破成功
+ **安全事件-创建用户账户（4720）、删除用户账户（4726）** + **安全事件-向本地组中添加成员（4732）、从本地组中删除成员（4733）** + **安全事件-允许侦听（5154）、允许网络连接（5156）**

网络连接追踪案例
  • 主机探测：目标端口为【0】
  • 协议类型为【1】-> ICMP协议

  • 安全事件-创建进程（4688）、退出进程（4689）
进程跟踪事件-案例

  • cmd.exe 进程ID：0x1c4
  • whoami.exe 创建者进程ID：0x1c4
  • ipconfig.exe 创建者进程ID：0x1c4

-> 在cmd中执行了whoami和ipconfig命令

  • 系统事件-事件日志服务启动（6005）、事件日志服务停止（6006）→ 开机、关机

Windows日志审计方式

Log Parser日志审计工具：可以分析操作系统的事件日志、注册表、文件系统、Active Directory、IIS。它可以像使用 SQL 语句一样查询分析这些数据，甚至可以把分析结果以各种图表的形式展现出来。

Event Log Explorer：它可以帮助用户深入分析日志事件、过滤特定事件、生成报告，并支持批量处理多个日志文件。相较于 Windows 内置的事件查看器，Event Log Explorer 提供了更多筛选、排序、导出和自动化任务的功能。

Linux日志分析

常用的Linux日志

登录信息

  • /var/log/lastlog：lastlog日志主要记录每个用户最后的登录信息

每个用户登录/注销、系统启动/停机

  • /var/log/wtmp：wtmp记录了系统所有的登录和注销，系统启动和停机活动

当前登录的用户信息

  • /var/run/utmp：记录了当前系统上所有已登录的用户信息。

登陆失败的信息

  • /var/log/btmp：用于记录所有失败的登录尝试。每当用户输入错误的用户名或密码，或因其他原因导致登录失败，系统都会在该文件中记录下相关信息，包括失败的用户名、尝试登录的时间、IP 地址等信息。

登录认证日志

  • 登录认证日志：通过分析/var/log/secure或/var/log/auth.log文件，我们可以得到登录信息，包含验证和授权方面的信息，sshd会将所有信息都记录其中，包括登录失败的信息。secure日志会随着时间为结尾名来进行分量保存，以防止文件过大。

    • /var/log/secure -> Redhat、CentOS

    • /var/log/auth.log -> Debian、Ubuntu
