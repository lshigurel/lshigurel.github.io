PHP文件包含

文件包含函数

include

  • include()在包含的过程中如果出现错误，只生成警告，并且脚本会继续

require

  • require()在包含的过程中如果出现错误，会生成致命错误并停止脚本

include_once、require_once

  • include_once()与require_once()，如果文件已包含，则不会包含，以避免函数重定义

使用这 4 种函数包含一个新的文件时，只要文件内容符合 PHP 语法规范，任何扩展名都可以被 PHP 解析。当包含非 PHP 语法规范的源文件时，将会暴露其源代码，造成文件读取漏洞。

在写法上可以使用 include(‘xxx’); 也可以使用 include ‘xxx’;

PHP伪协议

名称 含义
file:// 访问本地文件系统
http:// 访问HTTP(s)网址
ftp:// 访问FTP(s) URLs
php:// 访问输入/输出流(I/O streams)
zlib:// 压缩流
data:// 数据(RFC 2397)
glob:// 查找匹配的文件路径

任意文件读取

当文件包含包含的文件没有被 php 标签包裹时，会直接显示包含文件的内容，否则会先被解析成 php 代码，再将运行结果进行输出。（不是根据后缀来判断是否为 php 代码，而是根据文件内容）

任意文件读写 - php://filter

php://filter 可以获取指定文件源码。当它与包含函数结合时，php://filter 流会被当作 php 文件执行。 所以我们一般对其进行编码，让其不执行，从而导致任意文件读取。

条件：allow_url_fopen=Off/On、allow_url_include=Off/On

名称 描述 备注
resource=<要过滤的数据流> 指定了你要筛选过滤的数据流 必须
read=<读链的筛选列表> 可以设定一个或多个过滤器名称，以管道符( )分割
write=<写链的筛选列表> 可以设定一个或多个过滤器名称，以管道符( )分割

过滤器应用：

字符串过滤器 作用
string.rot13 对字符串执行ROT13转换
string.toupper 转换为大写
string.tolower 转换为小写
string.strip_tags 去除html和php标记
转换过滤器 作用
convert.base64-encode & convert.base64-decode base64编码/解码
convert.quoted-printable-enconde & convert.quoted-printable-deconde Quoted-printable编码
压缩过滤器 作用
zlib.deflate和zlib.inflate 在本地文件系统中创建gzip兼容文件的方法，但不产生命令行工具如gzip的头和尾信息，只是压缩和解压数据流中的有效载荷部分
bzip2.compress和bzip2.decompress 同上，在本地系统中创建bz2兼容文件的方法

实例用法：

?file=php://filter/read=convert.base64-encode/resource=index.php

PHP代码执行 - php://input

php://input 可以获取 POST 的数据流。当它与包含函数结合时，php://input 流会被当作 php 文件执行。从而导致任意代码执行。这可以理解成远程文件包含漏洞（RFI），即 POST 传过去 PHP 代码，然后被包含并执行。

条件：allow_url_fopen=Off/On、allow_url_include=On

PHP代码执行 - data://

data:// 同样类似与 php://input，可以让用户来控制输入流。 当它与包含函数结合时，用户输入的data:// 流会被当作 php 文件执行。从而导致任意代码执行。

条件：allow_url_fopen=On、allow_url_include=On、php >= 5.2

PHP代码执行 - zip://

zip:// 可以访问压缩包里面的文件。当它与包含函数结合时，zip:// 流会被当作 php 文件执行。从而实现任 意代码执行。

只需要是 zip 的压缩包即可，后缀名可以任意更改。相同的类型的还有 zlib://和bzip2://。

条件：PHP>=5.2

在PHP=5.2只能使用绝对路径，PHP>=5.3后也可以使用相对路径

PHP代码执行 - phar://

phar:// 有点类似 zip:// 同样可以导致任意代码

条件： PHP>=5.3

PHP代码执行 - 远程文件包含

进行远程文件包含时，如果页面的内容为php代码，则会进行本地解析。

条件：allow_url_fopen=On、allow_url_include=On

利用session.upload_progress进行getshell

在php中session.use_strict_mode默认值为0。此时用户是可以自己定义Session ID的。比如，我 们在Cookie里设置PHPSESSID=YU22X，PHP将会在服务器上创建一个文件： /tmp/sess_YU22X”(路径不一定是/tmp下)。
